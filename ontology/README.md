# Bordeaux Digital Twin Ontology

`bordeaux_ontology.ttl` is a standard, machine-readable **OWL ontology** (Turtle
syntax) describing the cross-domain data model of the Bordeaux Metropole
digital twin: 10 domains, 28 semantic relationships (each with a confidence
level), and 6 causal chains linking them together.

It is generated by [`export_owl.py`](export_owl.py) directly from the
`DOMAIN_META`, `ONTO_RELATIONS` and `CAUSAL_CHAINS` data already used by the
Streamlit ontology page (`app_pages/ontology.py`), so it always stays in sync
with the source of truth. Regenerate it with:

```bash
python ontology/export_owl.py
```

## Namespace

All terms use the `bto:` prefix: `http://bordeaux-twin.org/ontology#`

## Structure

- **Domains** (`Mobility`, `Weather`, `AirQuality`, `Water`, `Energy`,
  `Buildings`, `Population`, `Environment`, `Sensors`, `OnlineDataSet`) are
  each modeled as an `owl:Class`.
- **Relationships** (e.g. `generates`, `dependsOn`, `influencedBy`) are each
  modeled as an `owl:ObjectProperty` with `rdfs:domain`/`rdfs:range` set to
  the two domain classes it connects, plus a direct triple between the two
  classes (e.g. `bto:Mobility bto:generates bto:AirQuality`) so it can be
  queried directly.
- Because a plain triple can't carry extra metadata, each relationship is
  also **reified** (`rdf:Statement`) to attach:
  - `bto:hasConfidenceLevel` — `"Strong"`, `"Medium"` or `"Weak"`
  - `rdfs:comment` — the human-readable explanation
- **Causal chains** are modeled as `bto:CausalChain` individuals, each linked
  via `bto:hasStep` (inverse: `bto:partOfCausalChain`) to an ordered sequence
  of `bto:ChainStep` individuals carrying `bto:stepDomain`, `bto:stepRole`,
  `bto:stepOrder` and a `rdfs:comment` description.

## Opening it in Protégé

1. Download [Protégé](https://protege.stanford.edu/) (free, cross-platform).
2. `File → Open...` and select `bordeaux_ontology.ttl` (Protégé
   auto-detects the Turtle format).
3. Explore classes under the **Classes** tab, object properties under
   **Object properties**, and individuals (causal chain steps) under
   **Individuals**.

## Querying it with SPARQL

You can run these directly inside Protégé's SPARQL Query tab, or with
`rdflib` in Python:

```python
from rdflib import Graph
g = Graph()
g.parse("bordeaux_ontology.ttl", format="turtle")
for row in g.query(sparql_query):
    print(row)
```

**1. All relationships where Mobility is the source domain:**

```sparql
PREFIX bto: <http://bordeaux-twin.org/ontology#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?relationLabel ?target
WHERE {
  ?prop rdfs:domain bto:Mobility ;
        rdfs:label ?relationLabel .
  bto:Mobility ?prop ?target .
}
```

**2. All relationships with `Strong` confidence, with their explanation:**

```sparql
PREFIX bto: <http://bordeaux-twin.org/ontology#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?subject ?predicate ?object ?explanation
WHERE {
  ?stmt a rdf:Statement ;
        rdf:subject ?subject ;
        rdf:predicate ?predicate ;
        rdf:object ?object ;
        bto:hasConfidenceLevel "Strong" ;
        rdfs:comment ?explanation .
}
```

**3. Ordered steps of the "Urban Traffic -> Air Pollution -> Public Health" causal chain:**

```sparql
PREFIX bto: <http://bordeaux-twin.org/ontology#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?order ?domain ?role ?description
WHERE {
  ?chain rdfs:label "Urban Traffic → Air Pollution → Public Health" ;
         bto:hasStep ?step .
  ?step bto:stepOrder ?order ;
        bto:stepDomain ?domainClass ;
        bto:stepRole ?role ;
        rdfs:comment ?description .
  ?domainClass rdfs:label ?domain .
}
ORDER BY ?order
```

## Validating the file

```bash
python -c "from rdflib import Graph; g = Graph(); g.parse('ontology/bordeaux_ontology.ttl', format='turtle'); print(len(g), 'triples parsed OK')"
```

`export_owl.py` also re-parses the file it writes as a self-check, printing
the triple count on every run.
